Change Data Capture (CDC) en SAP

Métodos CDC Nativos de SAP

ODP (Operational Data Provider)

Método recomendado por SAP. Soporta CDC para la mayoría de tablas. Compatible con licenciamiento.

Recomendado Tiempo Real

SAP SLT (System Landscape Transformation)

Replicación en tiempo real basada en triggers. Requiere licencia específica.

Licencia Requerida Tiempo Real

SAP Data Services (BODS)

CDC basado en timestamp, log-based CDC y source-based CDC. Herramienta ETL completa.

ETL Completo Batch/Near RT

Limitaciones y Consideraciones

Restricciones RFC

SAP ha prohibido el uso de RFC para extracción de datos por herramientas de terceros (excepto aplicaciones internas de SAP).

Acceso Directo a BD

El acceso directo a la base de datos puede violar las licencias de SAP y generar costos de acceso indirecto.

Rutas Seguras

  • • ODP (Operational Data Provider)
  • • OData Services
  • • CDS Views (Core Data Services)
  • • SAP-approved connectors

Licenciamiento SAP y Acceso Indirecto

Riesgos de Licenciamiento Indirecto

¿Qué es el Acceso Indirecto?

Cuando sistemas o usuarios no-SAP acceden a datos o funcionalidades de SAP sin tener licencias Named User apropiadas.

Ejemplos que generan costos:
  • • Aplicaciones web que consultan SAP
  • • APIs que exponen datos SAP
  • • Herramientas ETL accediendo directamente a BD
  • • Dashboards que muestran data SAP

Modelos de Licenciamiento para Acceso Indirecto

Named User Licensing

Cada usuario individual requiere licencia SAP ($1,500-15,000/año)

Digital Access Licensing

Licencia por documento/transacción procesada (~$10-50 por documento)

Estrategias de Compliance

Rutas Aprobadas por SAP

  • ODP (Operational Data Provider): Interface oficial para extracción de datos
  • OData Services: APIs REST nativas de SAP
  • CDS Views: Vistas de datos certificadas
  • SAP-certified Connectors: Herramientas pre-aprobadas

RISE with SAP

El modelo RISE incluye ciertos derechos de acceso indirecto, pero requiere gestión cuidadosa de los límites.

Negociación Proactiva

Involucrar a SAP desde el diseño del proyecto para acordar modelos de licenciamiento antes de la implementación.

Conclusiones y Recomendaciones

Recomendaciones Estratégicas

1. Comenzar con ODP + AWS Glue

La combinación más segura desde perspectiva de licenciamiento y con menor riesgo técnico.

2. Implementación Iterativa

Iniciar con casos de uso críticos y expandir gradualmente para minimizar riesgo y acelerar time-to-value.

3. Involucrar SAP Licensing Temprano

Negociar términos de acceso indirecto antes de la implementación para evitar sorpresas costosas.

4. Invertir en Capacitación

Desarrollar capacidades internas para reducir dependencia de consultores externos a largo plazo.
